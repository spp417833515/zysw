# 收支数据流审计报告

> 审计日期: 2026-04-03

## 一、数据全景

```
Transaction 表（所有收支的唯一真相源）
├── type: income / expense / transfer
├── amount: 金额
├── payment_confirmed: 是否已实际收/付款
├── payment_account_type: company(公司) / personal(个人代付)
└── category_id → 关联分类

SalaryRecord 表（工资专用）
├── net_salary: 应发金额（计算值）
├── tax: 代扣个税
└── transaction_id → 关联到实际支付交易
```

## 二、每分钱的流向

```
全部收入 101,005.00
├── 已收款(confirmed=True)   101,005.00  → 进账户余额
└── 未收款(confirmed=False)       0.00  → 进应收账款

全部支出 13,963.35
├── 已付款(confirmed=True)   13,919.95  → 从账户扣减
│   ├── 工资交易              11,376.67  ← salary_records 关联
│   └── 其他交易               2,543.28  ← 办公/运营/招待等
└── 未付款(confirmed=False)      43.40  → 进应付（职工薪酬）

账户余额验算:
  0.00 + 101,005.00 - 13,919.95 = 87,085.05 ✅
```

## 三、各报表取数口径

### 正确的取数规则

| 报表 | 收入取什么 | 支出取什么 | 口径 |
|:---|:---|:---|:---|
| **利润表** | 全部收入 | 全部支出 | 权责发生制 |
| **现金流量表** | confirmed=True 的收入 | confirmed=True 的支出(排除工资交易) + salary_records关联交易金额 | 收付实现制 |
| **资产负债表** | — | — | 时点余额 |
| **趋势/分类报表** | 全部收入 | 全部支出 | 权责发生制 |
| **应收账款** | confirmed=False 的收入 | — | 未收款 |
| **应付账款** | — | confirmed=False 的支出(排除工资类) | 未付款 |
| **预算** | — | 全部该分类支出 | 承诺制 |
| **仪表盘** | 全部收入 | 全部支出 | 权责发生制 |

### 资产负债表取数

```
资产方:
  货币资金           = accounts.balance (实际余额)
  应收账款           = income & confirmed=False 的 SUM

负债方:
  应付账款           = expense & confirmed=False 的 SUM (排除工资类)
  应付职工薪酬       = ①完全未发工资 + ②少付差额 + ③员工垫付
    ① salary_records WHERE transaction_id IS NULL → SUM(net_salary)
    ② salary_records JOIN transactions → SUM(net_salary) - SUM(txn.amount)
    ③ 员工工资分类 & confirmed=False 的交易(排除工资发放关联)

所有者权益:
  未分配利润         = 资产合计 - 负债合计
```

## 四、关键防重规则

### 工资不重复计算
```
利润表: 查全部 expense (含工资交易) → 工资算一次 ✅
现金流量表:
  "支付其他"  → 排除 salary_records 关联的交易 (exclude_salary=True)
  "职工薪酬"  → 单独取 salary_records 关联交易的 amount
  两者不交叉 → 工资只算一次 ✅
```

### 现金流 vs 利润表 区别
```
利润表:   全部交易 (不管 payment_confirmed)
现金流:   只取 payment_confirmed=True (cash_basis=True)
两套独立数据源，互不干扰 ✅
```

### 应付职工薪酬 vs 应付账款 不重复
```
应付账款:     expense & confirmed=False & 分类名不含"工资"
应付职工薪酬: 工资欠款（三部分）
两者通过分类名排除，不会重叠 ✅
```

## 五、已修复的问题清单

| # | 问题 | 影响 | 修复方式 |
|:--|:---|:---|:---|
| 1 | 现金流量表用权责发生制数据 | 净额偏离实际余额 | 新增 cash_basis 参数，只查 confirmed=True |
| 2 | 工资在现金流中被双重计算 | 支出多算一倍工资 | exclude_salary 排除关联交易 |
| 3 | 现金流工资金额用 salary_records.net_salary | 与实际支付金额差 90 | 改用关联 transaction.amount |
| 4 | 应付职工薪酬只查当月未发放 | 漏掉少付差额+垫付 | 改为累计三维度查询 |
| 5 | 应付账款含工资类交易 | 与应付职工薪酬重复 | 排除工资分类 |

## 六、注意事项

1. **新增费用分类时**：如果分类名包含"工资"二字，会被自动归入"应付职工薪酬"而非"应付账款"
2. **salary_records.net_salary 与 transaction.amount 不一致**：这来自 actualPaid 手动调整。利润表用 salary_records（应发），现金流用 transaction（实付），差额进应付职工薪酬
3. **个人代付(personal)**：不影响公司账户余额，通过报销批次流程单独处理
